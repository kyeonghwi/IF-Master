## Key Architecture Patterns

### Idempotent Retry (Headline Feature)
1. Client calls `POST /api/retry/{log_id}`
2. Router fetches log row with `SELECT ... FOR UPDATE` (row lock)
3. Calls `GET /mock/status?key={idempotency_key}` to check external state
4. If already processed → returns `ALREADY_PROCESSED` without re-sending
5. Otherwise → calls `POST /mock/call`, writes result, appends to `audit_log`

### SSE Broadcast Fan-Out
- `app.state.client_queues` — set of `asyncio.Queue` (one per connected browser)
- `broadcast_event(event_type, data)` — pushes to all queues
- Scheduler and retry handler call `broadcast_event` on state changes
- `GET /api/stream` — streams from the queue; browser uses `useSSE` hook

### APScheduler In-Process Demo Traffic
- `services/scheduler.py` — fabricates transactions every 30s for live dashboard demo
- Calls `broadcast_event` to update connected clients

## Authentication Boundary

| Route | Auth Required | Reason |
|-------|---------------|--------|
| `/auth/*` | No | Login/logout endpoints |

| `/api/stream` | No | EventSource can't set headers |
| `/mock/*` | No | Internal harness |

## Known Architectural Notes

- **Ad-hoc schema migration in lifespan**: `main.py` runs `ALTER TABLE ... ADD COLUMN IF NOT EXISTS response_ms` on startup. Alembic is a declared dependency but not wired up — migration strategy should be formalized.
- **Alembic not activated**: `alembic` in `pyproject.toml` but no `alembic/` directory or `alembic.ini`. If the schema evolves, this gap will cause problems.
- **Auth bypass on `/api/stream`**: intentional (EventSource header limitation) but worth documenting in security reviews.
- **No frontend tests**: pytest covers the backend API surface; the frontend has no test setup (no Jest/Playwright config found).
